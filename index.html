2021

A mini college level project carried out using Matlab

Belgavi, Karnataka

Automatic cheque processing systems is one of the most studied areas where document analysis and biometrics data are involved. Especially in banks. The number of forgery cases are significantly increasing. The account holders signature can be easily forged.

To serve this purpose we have carried out a mini project where we have used a specific bank cheque leaf which acts as the input. The main features of the cheque are handwriting/thumb impression in which the name, signature, amount is mentioned by an individual. so, whenever a new account is opened, the bank stores all these sensitive details in their databases, which can be used as reference while detection of forgery

The next time when the customer arrives with a cheque leaf signed by himself or given by some other person, that image of cheque is scanned and stored. Further, pre-processing of the cheque is carried out by noise removal using 3x3 kernel median filter followed by smoothening of image. Then the RGB image is converted to gray scale and then to binary image with the help pf commands in Matlab. Segmentation is carried out where only the region of interest on the cheque are cropped using cropping points and command imcrop . For example if we are ffocused on signature, then it is extracted and compared with the original one stored in the database. As image is a matrix, image comparision is comparision of points in the matrix. If both matrix of reference image and new image are a match, then further procedure is carried out & if not then the transaction fails. The result is displayed as verification sucessfull or unsucessfull depending on the match.

2022

Developed a question answering system which semantically generates answers using GPT-3.

Belgavi, Karnataka

As it is evident that agriculture is the main occupation in India, relevant information if provided to the farmers, the boost in the yeild could be seen at higher scale.But due to lack of literacy and resources the farmers would opt for wrong practices.

This can be solved using recent technological advances in the feild of AI & ML. Our project titled "Text Data Analysis and modelling" is one such approach. It is a Q & A system that a farmers can make use of to solve their queries related to land, type of soil, fertilizer, climatic conditions,seeds,insects and diseases etc.

This Q & A system is built using a machine learning algorithm. To be more specific it uses OpenAI platform's Generative Pre-Training algorithm shortly named GPT. We have used GPT-3 algorithm as it was found to be comparitively more efficient and powerfull w.r.t. its predecessors GPT-2 and GPT-1. It is a massive language prediction and generation model capable of text generation which is human like. It does not require any fine-tuning and a few demonstration are sufficient for it to understand and perform desired task.

This project was initiated by collecting the data from agricultural websites.As a part of dataset creation we translated the data into english, framed question andd answer pairs that would be fed to the model.Tokenisation of the context was done and token length were generated. From the input context Q & A pairs were generated by model. The number of Q&A depends on the umber of tokens and length of each context. Using a function 'get_questions' and 'get_answers'. Next a discriminator model creates a file which has related and unrelated Q&A pairs. For the given questions, answers are retreived from the finetuned model by using the function 'answer_question'. If the relevant context is not found then it displays no context found for the asked question. We evaluated the modeel for 3 different test cases and drew conclusions on its efficiency

The model was tested for its execution by providing different number of context in batches of 100,500,1000,3000. It was observed that 100 and 500 were sucessfull in execution but 1000 and 3000 were not sucessfull in execution due to its increased token size. In order to execute them we required unlimited access of GPT3 or by paying for the use.

In second case, we train the model with context and observe how the model generates Q&A pairs. For every context multiple Q&A pairs were generated which were either reordered or twisted to get the maximum possible Q&A for each of the context.

Different contexts was given to the discriminator model in which 80% was used for training and 20% for testing. For example a set of 10 context was given 20 pairs that were generated where in context and 60 were out of context. For both the training and testing dataset, same pattern was followed as 1:3.
